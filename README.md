# Pelican/Pterodactyl Egg: Minecraft (Packwiz)

A [Pelican](https://pelican.dev)/[Pterodactyl](https://pterodactyl.io) egg for running Minecraft servers with [packwiz](https://packwiz.infra.link/) modpack management.

## Features

- Automatic packwiz installation on first setup
- Git-based modpack distribution (clone your config repo)
- Automatic mod updates before each server start (`packwiz update`)
- Supports Java 8, 17, and 21
- Configurable memory, ports, and JVM options

## How It Works

1. **Setup**: When you create a server, the egg installs packwiz and runs `packwiz install` to download all mods
2. **Startup**: Before each server start, it runs `packwiz update` to check for mod updates
3. **Runtime**: Standard Minecraft server with your packwiz-managed mods

## Quick Start

### Option A: Upload Files Manually

1. Create your `packwiz.toml` and mod references locally using the packwiz CLI
2. Upload `packwiz.toml` and the `mods/` directory via the panel's file manager
3. Start your server

### Option B: Use a Git Repository (Recommended)

1. Host your `packwiz.toml` and `mods/` directory in a git repository
2. When creating the server, set the **Packwiz Repo URL** variable to your repo
3. Set **Packwiz Repo Branch** if using a non-default branch
4. The egg will auto-clone and set everything up

## Installation in Pelican/Pterodactyl

1. Go to **Eggs > Egg Editor** in your panel admin
2. Click **Create New Egg**
3. Select **Import from JSON**
4. Upload `egg-minecraft-packwiz.json` or paste its contents
5. Configure your nest and save

## File Structure

Your server directory should contain:

```
server/
├── packwiz.toml          # Main config (REQUIRED)
├── mods/                 # Mod references (auto-generated by packwiz)
│   └── *.toml            # Individual mod entries
├── config/               # Optional: config overrides
│   └── *.toml            # Config file entries
├── assets/               # Optional: asset overrides
├── server.properties     # Standard Minecraft config
└── eula.txt              # EULA acceptance
```

## Egg Variables

| Variable | Default | Description |
|----------|---------|-------------|
| JAVA_VERSION | java21 | Java version (java8, java17, java21) |
| SERVER_PORT | 25565 | Port for players to connect |
| STARTER_MEM | 1024 | Initial heap size in MB |
| MAX_MEM | 4096 | Max heap size in MB |
| JAVA_OPTS | (G1GC flags) | Extra JVM arguments |
| PACKWIZ_REPO | (empty) | Git repo URL for packwiz config |
| PACKWIZ_BRANCH | main | Branch to use from the repo |

## Troubleshooting

### "packwiz install failed"
- Verify `packwiz.toml` has valid TOML syntax
- Check that your mod provider (CurseForge/Modrinth) settings are correct
- Ensure the server has internet access

### Mods not appearing after install
- Check the console output during setup for errors
- Verify `mods/` directory contains `.toml` reference files (not actual jar files)
- Restart the server to trigger `packwiz update`

### "No packwiz.toml found"
- Upload your `packwiz.toml` via the file manager before starting
- Or set the PACKWIZ_REPO variable to clone one
